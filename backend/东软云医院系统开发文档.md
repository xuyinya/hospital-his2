# 东软云医院系统开发文档

## 1. 项目概述

### 1.1 系统简介
东软云医院系统（HIS）是一个基于 B/S 架构的医院信息管理系统，实现医院门诊业务的信息化管理。

### 1.2 技术栈
| 层次 | 技术 | 版本 |
|------|------|------|
| 后端框架 | Spring Boot | 3.2.5 |
| ORM框架 | MyBatis-Plus | 3.5.5 |
| 数据库 | MySQL | 8.0 |
| JDK | Java | 21 |
| 连接池 | Druid | 1.2.21 |
| 接口文档 | Knife4j | 4.4.0 |
| 工具库 | Hutool | 5.8.25 |
| 认证 | JWT | 4.12.0 |

### 1.3 项目结构
```
backend/
├── src/main/java/com/neusoft/hospital/
│   ├── HospitalApplication.java          # 启动类
```

### 2.2 数据表结构

#### 患者表 (patient)
| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键，自增 |
| patient_name | VARCHAR(50) | 患者姓名 |
| gender | TINYINT | 性别（0女1男） |
| age | INT | 年龄 |
| id_card | VARCHAR(18) | 身份证号 |
| phone | VARCHAR(11) | 手机号 |
| address | VARCHAR(200) | 地址 |
| create_time | DATETIME | 创建时间 |
| update_time | DATETIME | 更新时间 |

```sql
CREATE TABLE patient (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    patient_name VARCHAR(50) NOT NULL,
    gender TINYINT DEFAULT 0,
    age INT,
    id_card VARCHAR(18),
    phone VARCHAR(11),
    address VARCHAR(200),
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

#### 科室表 (department)
| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键 |
| dept_name | VARCHAR(50) | 科室名称 |
| dept_code | VARCHAR(20) | 科室编码 |
| location | VARCHAR(100) | 科室位置 |
| status | TINYINT | 状态（0停用1启用） |

```sql
CREATE TABLE department (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    dept_name VARCHAR(50) NOT NULL,
    dept_code VARCHAR(20) UNIQUE,
    location VARCHAR(100),
    status TINYINT DEFAULT 1
);
```

#### 医生表 (doctor)
| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键 |
| doctor_name | VARCHAR(50) | 医生姓名 |
| dept_id | BIGINT | 所属科室ID |
| title | VARCHAR(20) | 职称 |
| specialty | VARCHAR(100) | 专长 |
| status | TINYINT | 状态 |

```sql
CREATE TABLE doctor (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    doctor_name VARCHAR(50) NOT NULL,
    dept_id BIGINT NOT NULL,
    title VARCHAR(20),
    specialty VARCHAR(100),
    status TINYINT DEFAULT 1,
    FOREIGN KEY (dept_id) REFERENCES department(id)
);
```

#### 挂号表 (registration)
| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键 |
| patient_id | BIGINT | 患者ID |
| doctor_id | BIGINT | 医生ID |
| dept_id | BIGINT | 科室ID |
| reg_type | VARCHAR(10) | 挂号类型（普通/专家） |
| reg_fee | DECIMAL(10,2) | 挂号费 |
| reg_time | DATETIME | 挂号时间 |
| status | TINYINT | 状态（0待诊1已诊2取消） |

```sql
CREATE TABLE registration (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    patient_id BIGINT NOT NULL,
    doctor_id BIGINT NOT NULL,
    dept_id BIGINT NOT NULL,
    reg_type VARCHAR(10) DEFAULT '普通',
    reg_fee DECIMAL(10,2),
    reg_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    status TINYINT DEFAULT 0,
    FOREIGN KEY (patient_id) REFERENCES patient(id),
    FOREIGN KEY (doctor_id) REFERENCES doctor(id),
    FOREIGN KEY (dept_id) REFERENCES department(id)
);
```

#### 病历表 (medical_record)
| 字段 | 类型 | 说明 |
|------|------|------|

### 5.2 常用查询
```sql
-- 查询今日挂号
SELECT r.*, p.patient_name, d.doctor_name, dept.dept_name
FROM registration r
LEFT JOIN patient p ON r.patient_id = p.id
LEFT JOIN doctor d ON r.doctor_id = d.id
LEFT JOIN department dept ON r.dept_id = dept.id
WHERE DATE(r.reg_time) = CURDATE();

-- 查询患者病历
SELECT mr.*, p.patient_name, d.doctor_name
FROM medical_record mr
LEFT JOIN patient p ON mr.patient_id = p.id
LEFT JOIN doctor d ON mr.doctor_id = d.id
WHERE mr.registration_id = ?;

-- 查询处方及明细
SELECT pd.*, dr.drug_name, dr.specification, dr.unit
FROM prescription_detail pd
LEFT JOIN drug dr ON pd.drug_id = dr.id
WHERE pd.prescription_id = ?;
```
