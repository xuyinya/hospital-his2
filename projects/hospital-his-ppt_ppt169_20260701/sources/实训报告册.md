一、实训目的

本次实训旨在通过开发一套完整的医院门诊信息管理系统（HIS），将软件工程、前后端开发、数据库设计、系统测试等理论知识应用于实际项目中。具体目标如下：掌握基于 Spring Boot 和 Vue 3 的企业级应用开发流程；理解医院门诊业务全流程，包括挂号、就诊、病历、处方、检查检验、收费等环节的信息化管理方式；掌握 MyBatis 持久层框架的使用和 SQL 优化技巧；学习 JWT 认证机制在前后端分离架构中的实现方式；掌握单元测试的编写方法和 Mockito 模拟框架的使用；培养独立分析问题、定位缺陷并修复的能力。

硬件环境：处理器为 Intel Core 系列，内存 16GB 以上，硬盘 512GB 以上，配备网络连接。

三、系统概述

东软云医院系统（HIS）是一套基于 B/S 架构的医院门诊信息管理系统，覆盖门诊业务的全流程信息化管理。系统分为员工端和患者门户两大子系统。

员工端供医院内部人员使用，根据角色分为管理员和医生。管理员负责基础数据维护，包括科室、医生、患者、药品等信息的管理。医生使用接诊工作台进行日常诊疗操作，包括查看患者列表、填写病历、开具处方等。此外，员工端还提供挂号管理、检查检验管理、处置管理和收费管理等业务模块。

患者门户供就诊患者使用，患者凭身份证号和密码登录后，可进行自助挂号、查看个人挂号记录、查阅病历和处方信息。患者门户的推出减少了窗口排队压力，提升了就医体验。

系统技术栈采用前后端分离架构。后端使用 Java 21 和 Spring Boot 3.2.5，通过 MyBatis 与 MySQL 8.0 数据库交互，使用 JWT 进行身份认证。前端使用 Vue 3 和 Element Plus 构建用户界面，通过 Axios 调用后端 RESTful API。


四、需求分析

4.1 功能需求

系统需要支持以下核心功能模块。挂号管理模块支持新增挂号、修改挂号信息、更新挂号状态（待诊、已诊、取消），以及按患者、医生、状态等多条件分页查询。患者管理模块支持患者信息的新增、修改、删除和查询，支持按身份证号精确查找。医生管理模块支持医生信息的增删改查，按科室筛选，删除时需检查是否有关联的挂号记录。药品管理模块支持药品信息的增删改查，支持逻辑删除（停用），按名称模糊分页查询。病历管理模块支持病历的新增、编辑和查询，支持按挂号 ID 查询病历。处方管理模块支持处方的创建、明细添加、状态更新（确认取药），按患者或医生自动过滤数据。检查检验和处置模块支持记录的新增、修改、删除、结果录入和分页查询。收费管理模块支持收费记录的创建、支付确认和分页查询。科室管理模块支持科室列表查询和详情查看。首页仪表盘展示挂号总数、已就诊数、待诊人数和总收入等统计信息。

患者门户需要提供患者登录、首页就诊概况、自助挂号、我的挂号、我的病历、我的处方等功能。

4.2 非功能需求

系统需要保证数据的一致性和完整性，关键业务操作需要事务支持。挂号费用等敏感数据应由后端计算，防止客户端篡改。用户认证使用 JWT Token，密码使用 BCrypt 加密存储。系统需具备良好的可维护性，代码规范统一，注释完整。


五、系统设计

5.1 架构设计

系统采用前后端分离的 B/S 架构。前端运行在浏览器中，通过 HTTP 请求与后端交互。后端提供 RESTful API，处理业务逻辑并读写数据库。前后端通过 JSON 格式进行数据交换。

前端架构采用 Vue 3 组合式 API（Composition API），使用 Vue Router 进行路由管理，Pinia 进行状态管理，Element Plus 提供 UI 组件，Axios 封装 HTTP 请求，Vite 作为构建工具。

后端架构采用分层设计，包括 Controller 层（接收 HTTP 请求、参数校验、返回响应）、Service 层（业务逻辑处理、事务管理）、Mapper 层（数据访问、SQL 映射），以及 Entity 层（数据模型）、Config 层（系统配置）、Security 层（JWT 认证与拦截）。

数据库使用 MySQL 8.0，通过 MyBatis 进行 ORM 映射，Druid 作为数据库连接池。

5.2 数据库设计

数据库名称 hospital_his，共包含 13 张表。核心业务表包括 patient（患者）、department（科室）、doctor（医生）、registration（挂号）、medical_record（病历）、prescription（处方）、prescription_detail（处方明细）、drug（药品）、examination（检查）、laboratory（检验）、treatment（处置）、payment（收费）。系统用户表 sys_user 用于存储管理员和医生账户。

各表之间通过外键建立关联关系。registration 表关联 patient、doctor 和 department。medical_record 和 prescription 关联 registration、patient 和 doctor。prescription_detail 关联 prescription 和 drug。这种设计保证了数据的引用完整性，同时通过外键约束防止误删除操作。

5.3 模块划分

系统按照业务领域划分为以下模块。认证模块负责用户登录、JWT 令牌生成与验证、角色权限控制。首页仪表盘模块展示统计数据。挂号管理模块处理挂号全流程。患者管理模块维护患者档案。医生管理模块管理医生信息。药品管理模块管理药品目录。病历管理模块记录诊疗过程。处方管理模块管理用药方案。检查管理模块管理检查项目。检验管理模块管理检验项目。处置管理模块管理处置操作。收费管理模块处理费用结算。科室管理模块提供科室信息查询。患者门户模块为患者提供自助服务。


六、系统实现

6.1 后端实现

后端使用 Spring Boot 3.2.5 作为基础框架，按分层架构组织代码。Controller 层统一使用 RestController 注解，请求路径以 /api 开头，返回统一封装的 Result 对象，包含 code、message 和 data 三个字段。分页查询返回 PageResult 对象，包含 total 和 rows 两个字段。

JWT 认证通过 JwtUtil 工具类实现令牌的生成和解析，JwtInterceptor 拦截器拦截所有 /api 开头的请求（除登录接口和 Swagger 路径外），从请求头中提取 Token 并验证，解析出的用户信息存入 request 属性中供后续使用。密码使用 BCryptPasswordEncoder 进行加密存储。

Service 层负责业务逻辑处理，关键操作使用 @Transactional 注解保证事务一致性。Mapper 层使用 MyBatis 注解和 XML 映射文件两种方式，简单的 CRUD 使用注解，复杂的多表关联查询使用 XML。

6.2 前端实现

前端使用 Vue 3 组合式 API，每个页面组件使用 <script setup> 语法。路由分为员工端路由和患者门户路由，通过 beforeEach 导航守卫进行权限控制。Axios 封装在 utils/request.js 中，统一处理请求拦截（添加 Authorization 头）和响应拦截（错误提示和 401 跳转）。

页面组件遵循统一的布局风格，使用 Element Plus 的 el-card、el-table、el-form、el-dialog 等组件构建。搜索栏、分页、表格操作按钮等 UI 模式在各个管理页面中保持一致。

6.3 关键代码示例

统一返回结果封装：
public class Result<T> { private Integer code; private String message; private T data; public static <T> Result<T> success(T data) { ... } public static <T> Result<T> error(String message) { ... } }

JWT 令牌生成：
public String generateToken(Long userId, String username, String role, Long doctorId) { return Jwts.builder().subject(String.valueOf(userId)).claim("username", username).claim("role", role).claim("doctorId", doctorId).issuedAt(now).expiration(expiration).signWith(getSigningKey()).compact(); }

七、系统测试

7.1 测试概述

本次测试针对系统的后端核心业务模块进行单元测试，覆盖 JWT 安全模块、患者管理模块、药品管理模块和挂号管理模块，共计 30 个测试用例。JwtUtil 采用纯单元测试方式，不依赖 Spring 容器和数据库。Service 层使用 Mockito 框架模拟 Mapper 接口，不依赖真实数据库，验证业务逻辑的正确性。

7.2 测试执行结果

全部 30 个测试用例均已完成执行，通过 30 个，失败 0 个，通过率 100%。其中 JwtUtilTest 执行 5 个全部通过，DrugServiceTest 执行 7 个全部通过，PatientServiceTest 执行 8 个全部通过，RegistrationServiceTest 执行 10 个全部通过。

7.3 缺陷发现与修复

测试过程中共发现 4 个缺陷并全部修复。药品管理模块的修改接口存在字段缺失，DrugMapper.update 方法的 SQL 语句缺少 status 字段，导致停用操作无效。患者自助挂号的费用计算存在安全隐患，修复为后端根据挂号类型重新计算。两个 Service 实现类缺少 @Transactional 注解，补充后保证事务原子性。挂号 Mapper 的 insert 语句缺少主诉和现病史字段，补充后患者填写的症状信息可正确保存。


八、系统部署

8.1 后端部署

编译打包使用命令 mvn clean package -DskipTests，在 JDK 21 环境下执行。生成的 JAR 文件位于 target/hospital-his-1.0.0.jar。启动命令为 java -jar target/hospital-his-1.0.0.jar，服务运行在 8080 端口。API 文档可通过 http://localhost:8080/doc.html 访问。

8.2 前端部署

前端构建使用命令 npm run build，构建产物输出到 dist 目录。开发环境通过 npm run dev 启动，端口 3000，Vite 配置了代理将 /api 请求转发到后端 8080 端口。

8.3 数据库初始化

首次部署需执行 hospital_his.sql 脚本创建数据库和表结构，并插入初始化数据。数据库用户名 root，密码 12345，连接地址为 localhost:3306。

九、实训总结

本次实训完成了东软云医院系统（HIS）的开发、测试和部署全流程。通过本次实训，深入理解和掌握了以下内容。

在技术层面，掌握了 Spring Boot 3.2.5 和 Vue 3 的前后端分离开发模式，理解了分层架构的设计思想和各层的职责划分。学习了 MyBatis 框架的使用，包括注解方式和 XML 映射方式的适用场景。掌握了 JWT 认证机制在前后端分离项目中的实现方式，以及 BCrypt 密码加密的使用。学习了 JUnit 5 和 Mockito 框架的单元测试编写方法，理解了 Mock 测试在隔离外部依赖中的作用。

在业务层面，深入了解了医院门诊业务的全流程，包括挂号、就诊、病历记录、处方开具、检查检验、收费等环节的信息化处理方式，理解了不同角色（管理员、医生、患者）在系统中的权限和操作范围。

在工程实践层面，体验了从需求分析、系统设计、编码实现到测试部署的完整软件开发生命周期。通过修复测试中发现的缺陷，积累了定位和解决问题的实际经验。通过 Git 进行版本管理，了解了团队协作的基本流程。

本次实训也存在一些可以改进的方向。测试覆盖主要集中在后端 Service 层，后续可以增加 Controller 层的接口测试和前端组件的单元测试。系统的错误处理机制可以进一步完善，提供更友好的用户提示。系统的性能优化和安全性方面还有提升空间，如增加操作审计日志、接口访问限流等措施。
